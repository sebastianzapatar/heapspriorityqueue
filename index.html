Parte I — Fundamentos
01 — Definición

¿Qué es un Heap?

Un árbol binario completo que mantiene una relación de orden entre padres e hijos.

🌳

Árbol Binario Completo

Todos los niveles están llenos excepto posiblemente el último, que se llena de izquierda a derecha. Esto garantiza una representación eficiente por arreglo.

📏

Propiedad del Heap

Cada nodo padre tiene una relación de orden con sus hijos. En un Min-Heap, el padre ≤ hijos. En un Max-Heap, el padre ≥ hijos.

⚡

Acceso O(1) al extremo

La raíz siempre contiene el mínimo (Min-Heap) o el máximo (Max-Heap). Acceder a este valor es instantáneo.

💡 Concepto clave: Un heap NO es un árbol de búsqueda. No mantiene un orden total, solo garantiza que el elemento más prioritario está en la raíz. Los hermanos no tienen relación de orden entre sí.
Parte I — Fundamentos
02 — Tipos

Min-Heap vs Max-Heap

🔽 Min-Heap

Padre ≤ Hijos → La raíz es el mínimo.
Ideal para encontrar el elemento más pequeño rápidamente.

🔼 Max-Heap

Padre ≥ Hijos → La raíz es el máximo.
Ideal para encontrar el elemento más grande rápidamente.

Parte II — Operaciones
04 — Operaciones

Operaciones del Heap

⬆️ Insert (Sift Up)

  1. Agregar el nuevo elemento al final del arreglo.
  2. Comparar con su padre.
  3. Si viola la propiedad del heap, intercambiar con el padre.
  4. Repetir hasta llegar a la raíz o restaurar la propiedad.
Complejidad: O(log n) — sube como máximo la altura del árbol.

⬇️ Extract (Sift Down)

  1. Guardar el valor de la raíz (min o max).
  2. Reemplazar la raíz con el último elemento.
  3. Comparar con sus hijos.
  4. Intercambiar con el hijo menor (Min-Heap) o mayor (Max-Heap).
  5. Repetir hasta restaurar la propiedad.
Complejidad: O(log n) — baja como máximo la altura del árbol.

10 — Animación

Inserción: Heaps vs BST

Observa cómo se inserta un nuevo valor en cada estructura (Sift-Up en Heaps vs Descenso en BST).

Min-Heap: 20 se inserta al final. Como 20 < 40, sube (Sift-Up) y se intercambia. Luego 20 > 10, por lo que se detiene.
Max-Heap: 80 se inserta al final. Como 80 > 50, sube (Sift-Up) y se intercambia. Luego 80 < 90, por lo que se detiene.
BST: 55 baja comparando desde la raíz. >40 (Der), <60 (Izq), >50 (Der). Se inserta ahí (no sube).

¿Por qué buildHeap es O(n) y no O(n log n)?
Aunque insertar un elemento es O(log n), el algoritmo heapify bottom-up procesa los nodos desde las hojas hacia la raíz. La mayoría de los nodos están en los niveles inferiores y requieren poco trabajo. La suma total es una serie geométrica que converge a O(n).

12 — Comparación

Heap vs BST

¿Cuándo usar cada uno? Son estructuras complementarias con propósitos diferentes.

Característica Heap BST
Propiedad de orden Padre ≤ / ≥ hijos Izquierdo < Padre < Derecho
Estructura Árbol binario completo No necesariamente completo
Representación Array (sin punteros) Nodos con punteros izq/der
Acceso rápido a Min / Max → O(1) Cualquier → O(log n)
Búsqueda arbitraria O(n) O(log n)
Inserción O(log n) O(log n) promedio, O(n) peor
Eliminación del extremo O(log n) O(log n)
Orden total ❌ No mantiene ✅ In-order traversal
Uso principal Colas de prioridad, HeapSort Búsquedas, recorridos ordenados
⚡ La diferencia clave: Un Heap optimiza el acceso al mínimo/máximo. Un BST optimiza búsquedas de cualquier valor. Si solo necesitas "¿cuál es el más prioritario?", usa un Heap.
Parte IV — Heaps en Java
13 — Implementación

MinHeap Genérico

Implementación desde cero usando generics y Comparable<T>.

MinHeap.java 
public class MinHeap<T extends Comparable<T>> {

    private final List<T> heap;

    // Insertar: agregar al final + sift up — O(log n)
    public void insert(T value) {
        heap.add(value);
        siftUp(heap.size() - 1);
    }

    // Extraer el mínimo: raíz ← último elemento + sift down — O(log n)
    public T extractMin() {
        T min = heap.get(0);
        T last = heap.remove(heap.size() - 1);
        if (!heap.isEmpty()) {
            heap.set(0, last);
            siftDown(0);
        }
        return min;
    }

    // Sift Up: subir mientras sea menor que el padre
    private void siftUp(int index) {
        while (index > 0) {
            int parent = (index - 1) / 2;   // fórmula del padre
            if (heap.get(index).compareTo(heap.get(parent)) < 0) {
                swap(index, parent);
                index = parent;
            } else break;
        }
    }

    // Sift Down: bajar buscando el menor de los hijos
    private void siftDown(int index) {
        int size = heap.size();
        while (true) {
            int smallest = index;
            int left = 2 * index + 1, right = 2 * index + 2;

            if (left < size && heap.get(left).compareTo(heap.get(smallest)) < 0)
                smallest = left;
            if (right < size && heap.get(right).compareTo(heap.get(smallest)) < 0)
                smallest = right;

            if (smallest != index) {
                swap(index, smallest);
                index = smallest;
            } else break;
        }
    }
}
🔑 Para convertir en MaxHeap: solo cambia la dirección de compareTo.
Donde dice < 0 cámbialo por > 0.

15 — Max-Heap

Implementar Max-Heap

3 formas de crear un Max-Heap con PriorityQueue.

Opción 1 — Collections.reverseOrder() 
import java.util.Collections;

// Forma más común y legible
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(
    Collections.reverseOrder()
);

maxHeap.offer(10);
maxHeap.offer(50);
maxHeap.offer(30);

System.out.println(maxHeap.poll());  // 50 ← máximo
Opción 2 — Comparator.reverseOrder() 
import java.util.Comparator;

// Equivalente, usando Comparator directamente
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(
    Comparator.reverseOrder()
);

maxHeap.offer(10);
maxHeap.offer(50);
maxHeap.offer(30);

System.out.println(maxHeap.poll());  // 50 ← máximo
Opción 3 — Lambda 
// Lambda: invertir la comparación
// ⚠️ b - a puede desbordar con valores extremos;
//    es más seguro usar Integer.compare(b, a)
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(
    (a, b) -> Integer.compare(b, a)
);

maxHeap.offer(10);
maxHeap.offer(50);
maxHeap.offer(30);

System.out.println(maxHeap.poll());  // 50 ← máximo

// ⚠️ Con objetos personalizados:
PriorityQueue<Tarea> cola = new PriorityQueue<>(
    Comparator.comparingInt(Tarea::prioridad).reversed()
);
💡 Resumen: Las tres formas producen el mismo resultado. Collections.reverseOrder() es la más legible. Para objetos personalizados, usa Comparator.comparingInt(...).reversed().

16 — Objetos

Heaps con Objetos

El heap no solo ordena números: cualquier clase puede priorizarse con Comparable o un Comparator.

🅰️ Con Comparable

Paciente.java 
// La clase define su "orden natural"
public class Paciente implements Comparable<Paciente> {
    private final String nombre;
    private final int gravedad; // 1 (leve) a 10 (crítico)

    @Override
    public int compareTo(Paciente otro) {
        // Compara por gravedad → el heap decide quién sube
        return Integer.compare(this.gravedad, otro.gravedad);
    }
}
Ventaja: el orden viaja con la clase. Un MaxHeap<Paciente> funciona sin configuración extra.

🅱️ Con Comparator

PriorityQueueExamples.java 
// record: clase inmutable en una línea
record Tarea(String nombre, int prioridad) {}

// Min-Heap: menor número = más urgente
PriorityQueue<Tarea> cola = new PriorityQueue<>(
    Comparator.comparingInt(Tarea::prioridad)
);

cola.offer(new Tarea("Enviar correo", 3));
cola.offer(new Tarea("Bug crítico", 1));

cola.poll(); // → "Bug crítico" (prioridad 1)

// Max-Heap: basta con .reversed()
PriorityQueue<Tarea> inversa = new PriorityQueue<>(
    Comparator.comparingInt(Tarea::prioridad).reversed()
);
Ventaja: cada cola puede tener su propio criterio sin tocar la clase.
Parte V — Algoritmos
17 — HeapSort

HeapSort paso a paso

Ordenamiento in-place en O(n log n): construye un Max-Heap y extrae el máximo repetidamente.

📜 Algoritmo

  1. Construir el Max-Heap: aplicar heapify desde el último nodo no-hoja (n/2 − 1) hasta la raíz. Costo: O(n).
  2. Intercambiar: mover la raíz (el máximo) al final del arreglo.
  3. Reducir el heap: el último elemento ya está ordenado; se excluye del heap.
  4. Heapify de la raíz: restaurar la propiedad del Max-Heap.
  5. Repetir los pasos 2–4 hasta que el heap quede vacío.
💡 In-place: no necesita memoria extra — el mismo arreglo actúa como heap y como zona ordenada.
HeapSort.java 
public void sort(int[] arr) {
    int n = arr.length;

    // 1. Construir Max-Heap — O(n)
    for (int i = n / 2 - 1; i >= 0; i--)
        heapify(arr, n, i);

    // 2. Extraer el máximo uno por uno
    for (int i = n - 1; i > 0; i--) {
        // Raíz (máximo) → al final
        int temp = arr[0];
        arr[0] = arr[i];
        arr[i] = temp;

        // Restaurar heap en la parte no ordenada
        heapify(arr, i, 0);
    }
}

// Hunde el nodo i hasta cumplir la propiedad Max-Heap
void heapify(int[] arr, int n, int i) {
    int largest = i;
    int left = 2 * i + 1, right = 2 * i + 2;

    if (left < n && arr[left] > arr[largest])  largest = left;
    if (right < n && arr[right] > arr[largest]) largest = right;

    if (largest != i) {
        int swap = arr[i];
        arr[i] = arr[largest];
        arr[largest] = swap;
        heapify(arr, n, largest); // seguir bajando
    }
}

ColaUrgencias.java — núcleo del sistema 
public class ColaUrgencias {

    // Criterio de atención:
    //   1. Mayor gravedad primero (Max-Heap)
    //   2. A igual gravedad → el que llegó primero (FIFO)
    private static final Comparator<Paciente> ORDEN_ATENCION =
            Comparator.comparingInt(Paciente::getGravedad).reversed()
                      .thenComparing(Paciente::getHoraLlegada);

    private final PriorityQueue<Paciente> cola;

    public ColaUrgencias() {
        this.cola = new PriorityQueue<>(ORDEN_ATENCION);
    }

    // Registrar paciente — O(log n)
    public void registrarPaciente(Paciente p) { cola.offer(p); }

    // Atender al MÁS GRAVE — O(log n)
    public Paciente atenderSiguiente() { return cola.poll(); }

    // Consultar el siguiente sin atenderlo — O(1)
    public Paciente verSiguiente() { return cola.peek(); }
}
🚨 El momento clave de la demo: con 4 pacientes ya en espera, llega "Diego Ramírez" con paro cardiorrespiratorio (gravedad 10). Un solo offer() y el heap lo sube a la raíz en O(log n): será el siguiente en ser atendido, sin reordenar ninguna lista manualmente.
⚠️ ¿Por qué el desempate? PriorityQueue no es estable: con prioridades iguales no garantiza orden de llegada. Por eso el comparador agrega .thenComparing(Paciente::getHoraLlegada) — en la demo, Ana (07:45) y Elena (08:50) tienen gravedad 5, y Ana se atiende primero por haber llegado antes.
